# 博客主题发现 (Blog Topic Discover)

## Who
希望从近期工作中提取有价值的学习经验、试错过程或新颖方法以撰写技术博客的开发人员和工程师。

## What
通过分析 git 提交、文件修改以及至关重要的 **智能体与用户/LLM之间的对话记录**，实现“从起点到终点”回顾近期完成的任务轨迹。它会评估在此过程中双方是如何通过交流来理解并最终解决问题的，极其关注如何有效地验证了问题已被解决。同时识别为了达成目标所做的更改，梳理正确/错误的决策，强调应用了哪些新颖想法或方法，并提出引人注目的技术博客主题方案。

## When
- 用户宣布一个任务或里程碑已经完成。
- 用户询问“关于这部分工作我能写些什么？”或“给我一些博客灵感”。
- 用户在编码会话结束后明确调用 `blog-topic-discover` 技能。

## Where
- 分析结果可以直接输出到对话中，或者根据用户偏好保存为工作区中的 markdown 总结文件（如 `blog_ideas.md`）。

## Why
- 将日常的编码挑战和踩坑经验转化为可共享的知识，有助于建立个人品牌和充实团队文档。
- 刚完成任务时，对于“为什么这样决策”的上下文记忆是最鲜活的。
- 挖掘 git 历史和聊天记录可以揭示比仅仅看最终代码更深刻的见解。

## Inputs
- **时间范围 / 起点**（必填）：提交哈希值、时间戳或逻辑起点（例如“从昨天开始”、“最近的5次提交”、“本次对话”）。
- **最终目标**（选填）：最初想要达到的目标是什么。

## Output
使用以下特定的艺术模板生成一篇完整且结构优雅的博客草稿：

```markdown
# <引人入胜的博客标题>

**【缘起性不空】(Origin)**
<任务的背景、为什么开始这项工作以及最初的目标。>

**【臆想不糊象】(Imagination)**
<对解决方案的最初设想，以及在遭遇现实碰壁前所做的假设。>

**【问题心中起】(Inquiry)**
<遇到的主要问题、碰到的 bug 或需要解决的难题。智能体与用户之间的对话是如何弄清真正的问题所在的。>

**【求解不甚解】(Exploration)**
<协作与挣扎的过程。一来一回的交流、被撤回的错误决定，以及最终如何达成并有效验证解决方案的过程。>

**【反思更人类】(Reflection)**
<“更具人性”的思考与总结。我们为什么会犯那些错误？这反映了我们的工作流或思维方式存在什么漏洞？>

**【洞见显智慧】(Insight)**
<学到的核心技术或方法论教训。应用了哪些新奇的工具、方法或架构模式。>
*(必须在此处包含一个 `mermaid` 图表，以直观地解释核心逻辑、流程或架构变更，便于读者轻松理解)*

**【新问开新局】(Evolution)**
<这引发了什么新问题，或是下一步合乎逻辑的计划是什么。>

—— by 术子米德@<当前日期，如：2026年x月z日>
（备注：问题是真的，过程也是真的，内容都是生成的）
```

## Constraints
- 调查结果必须严格基于提供的 git diff、提交记录和最近的对话记录。请勿捏造挑战。
- 重点关注“旅程”（即“原因”和“方法”），而不是仅仅列出枯燥的变更日志（即“做了什么”）。
- 对错误或不完美的决策保持诚实——这往往能写出最好的博客文章。
- **可视化支持**：务必包含至少一个 `mermaid` 图表，让复杂的解决路径或架构变得清晰易懂。

## One More Thing
如果要回顾的时间范围或起点不明确，在继续操作前，请先向用户询问需要往回追溯多久。

## How

### Phase 1: Discovery & Context Gathering (发现与收集上下文)
1. 询问或确定回顾的起点（例如特定的 commit，或是对话的开始）。
2. 收集日志：使用 git 工具（`git log`、`git diff`）或查阅对话历史，重构从起点到终点的时间线。
3. 明确最初的目标与最终的结果。

### Phase 2: Analysis & Extraction (分析与提取)
1. 提取 **历程 (Journey)**：主要里程碑或方向调整是什么？
2. 提取 **决策与协作 (Decisions & Collaboration)**：方法在哪里发生了改变？对话历史是如何塑造对问题的最终理解的？遇到了什么 bug，双方是如何协作解决的，以及解决方案是如何被有效验证的？是否有必须撤销的“错误”决定？
3. 提取 **新颖点 (Novelty)**：在这里独特地应用了哪些新技术、工具或方法？

### Phase 3: Brainstorming & Output (头脑风暴与输出)
1. 综合提取的分析内容和历程记录。
2. 严格使用提供的艺术博客模板（【缘起性不空】、【臆想不糊象】等），撰写一篇完整且引人入胜的博客草稿，重点突出问题解决与协作的过程。
3. 将起草好的博客文章展示给用户供其审查。
